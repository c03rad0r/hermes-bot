# PLAN: OpenWrt Custom Repository for TollGate

## Problem
`configurationwizzard` declares `DEPENDS:=+tollgate-wrt +nodogsplash +luci +jq`, but `tollgate-wrt` isn't in the official OpenWrt package feed. This means:
- `opkg install configurationwizzard` → fails with "Unknown package 'tollgate-wrt'"
- Endo has to manually download + install tollgate from blossom URL
- No automatic upgrade path for tollgate when net4sats is updated

## Current State

| Repo | Makefile Status | Artifact Location |
|---|---|---|
| `tollgate-module-basic-go` | Has packaging Makefile (SDK-cross-compiled Go binaries) | `releases.tollgate.me` via blossom |
| `configurationwizzard` | Has packaging Makefile, declares `DEPENDS:=+tollgate-wrt` | GitHub Releases + blossom |

The packaging Makefiles already work at **build time** (when both feeds are `src-link`'d into the SDK). The gap is **runtime**: the router's opkg/apk doesn't know where to find tollgate-wrt.

## Recommended Approach: Hybrid Custom Repository

### Option A: Hosted Package Index on releases.tollgate.me (Recommended)

The release manager at `releases.tollgate.me` already publishes `.ipk`/`.apk` files. Adding `Packages.gz` (opkg) and `packages.adb`/`APKINDEX.tar.gz` (apk) index files turns it into a proper package repository.

**How it works:**

```
releases.tollgate.me/
  alpha/
    v0.5.0-alpha3/
      aarch64_cortex-a53/
        1fcc1635...71d.apk
        Packages.gz         ← NEW: opkg index
        packages.adb        ← NEW: apk index
      mipsel_24kc/
        2fb588b10...82.ipk
        Packages.gz         ← NEW: opkg index
```

### Option B: GitHub Pages Feed on net4sats.github.io (Fallback)

If `releases.tollgate.me` is hard to modify (JavaScript SPA), host the feed metadata on `net4sats.github.io`:

```
docs.net4sats.cash/
  feeds/
    v0.5.0/
      aarch64_cortex-a53/Packages.gz
      mipsel_24kc/Packages.gz
```

The actual `.ipk`/`.apk` files stay on blossom/releases.tollgate.me — the index just tells the package manager where to download them.

### Option C: Post-Inst Script (Quick-and-Dirty)

Add `postinst` to `configurationwizzard.ipk` that runs:
```sh
#!/bin/sh
[ -z "${IPKG_INSTROOT}" ] || exit 0  # skip during build-time install

# Add tollgate feed
if command -v apk >/dev/null 2>&1; then
    # OpenWrt 25.x (apk)
    mkdir -p /etc/apk/repositories.d/
    echo "https://releases.tollgate.me/alpha/v0.5.0-alpha3/aarch64_cortex-a53/packages.adb" \
      > /etc/apk/repositories.d/tollgate.list
    apk update
else
    # OpenWrt 24.x (opkg)
    echo "src/gz tollgate https://releases.tollgate.me/alpha/v0.5.0-alpha3/aarch64_cortex-a53/Packages.gz" \
      >> /etc/opkg/customfeeds.conf
    opkg update
fi

# Now opkg/apk can resolve DEPENDS:=+tollgate-wrt
```

## Recommended Path: Option A + Option C (Combined)

**Phase 1: Generate Package Index (Options A)**

1. Write `scripts/generate-opkg-index.sh` — uses `opkg-make-index` or manual `gzip` of a generated `Packages` file
2. Write `scripts/generate-apk-index.sh` — uses `apk index` to generate `APKINDEX.tar.gz`
3. Generate indices for existing v0.5.0-alpha3 artifacts
4. Upload indices alongside `.ipk`/`.apk` to blossom or releases.tollgate.me

**Phase 2: Postinst script (Option C)**

1. Update `configurationwizzard/packaging/files/postinst` to add the custom feed
2. Ensure it handles both opkg and apk paths
3. Ensure `IPKG_INSTROOT` guard for build-time installs
4. Ensure it checks if feed already exists (idempotent)

**Phase 3: Full Custom Feed (Long-Term)**

1. Revisit the stalled feed PR for tollgate-module-basic-go
2. Set up automated CI that builds + publishes both the package AND the index
3. Consider `src-git` feed for OpenWrt SDK (build-time)

## Why NOT Other Approaches

| Approach | Problem |
|---|---|
| **Post-install curl download** | Fragile, no upgrade/rollback, breaks opkg transaction guarantees, bad practice per OpenWrt forums |
| **Bundle .ipk inside .ipk** | Can't declare proper DEPENDS, bloats the package, license issues |
| **Full custom feed PR** | Stalled for months, needs reviewer bandwidth, touches many files |
| **Static feed file in repo** | Requires CI to regenerate on every release |

The hybrid approach (index + postinst) gives us:
- `opkg install configurationwizzard` → automatically resolves `tollgate-wrt` from the custom feed
- `opkg upgrade` works for both packages
- No OpenWrt upstream review needed
- Can ship this week

## Implementation Details

### Package Index Format (opkg)

```makefile
# Generated by scripts/generate-packages-index.sh
Package: tollgate-wrt
Version: 0.5.0-alpha3
Depends: nodogsplash, luci, jq
Filename: 2fb588b10a445555923f1325d11c3cb28220cb32f078b283d71d4d3100e58286.ipk
Size: 1234567
SHA256sum: 2fb588b10a445555923f1325d11c3cb28220cb32f078b283d71d4d3100e58286
```

### Package Index Format (apk)

```bash
# apk index -o APKINDEX.tar.gz *.apk
# Then sign with:
# apk sign --key /path/to/tollgate.key APKINDEX.tar.gz
```

### Postinst Script

```sh
#!/bin/sh
# configurationwizzard postinst — register tollgate feed
[ -z "${IPKG_INSTROOT}" ] && exit 0

FEED_URL="https://releases.tollgate.me/alpha/v0.5.0-alpha3"

if command -v apk >/dev/null 2>&1; then
    # OpenWrt 25.x
    mkdir -p /etc/apk/repositories.d/
    if ! grep -q "releases.tollgate.me" /etc/apk/repositories.d/*.list 2>/dev/null; then
        echo "${FEED_URL}/aarch64_cortex-a53/packages.adb" \
          > /etc/apk/repositories.d/tollgate.list
        apk update
    fi
else
    # OpenWrt 24.x
    if ! grep -q "releases.tollgate.me" /etc/opkg/customfeeds.conf 2>/dev/null; then
        echo "src/gz tollgate ${FEED_URL}/mipsel_24kc/Packages.gz" \
          >> /etc/opkg/customfeeds.conf
        opkg update
    fi
fi
```
